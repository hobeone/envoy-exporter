# Product Guidelines

## Prose Style
- **Concise and technical:** Documentation and communication should focus on facts, implementation details, and clear technical specifications.

## Brand Messaging & Terminology
- **Domain Accuracy:** Consistently use Enphase-specific terminology (e.g., Envoy, Inverter, Production, Consumption) to ensure alignment with the hardware source.

## Visual Identity
- **Professional and standardized:** Adhere to standard Go logging practices and clean Markdown for documentation. Avoid unnecessary decoration.

## User Interface & Experience
- **Observability:** Prioritize machine-readable logs (JSON) to facilitate automated analysis and monitoring.
- **Error Handling:** Provide descriptive error messages that include actionable steps for the user to resolve issues.
